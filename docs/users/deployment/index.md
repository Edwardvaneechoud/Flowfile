# Deployment

Choose how to run Flowfile based on your needs.

| Method | Best For | Multi-user | Installation |
|--------|----------|------------|--------------|
| [Browser (WASM)](browser.md) | Trying Flowfile, quick tasks | No | None |
| [Desktop App](desktop.md) | Getting started, local development | No | Download |
| [Python Package](python.md) | Scripting, CI/CD, automation | No | pip install |
| [Docker](docker.md) | Teams, servers, production | Yes | docker compose |

## Quick Comparison

### Browser (WASM)
[**Try it now →**](https://demo.flowfile.org) Zero installation. Opens in your browser with 14 essential nodes. Data stays 100% local. Great for quick exploration or when you can't install software.

### Desktop App
Download and run. No setup required. Best for exploring Flowfile or building flows locally on macOS or Windows.

### Python Package
`pip install flowfile`. Run flows programmatically or integrate into existing Python projects.

### Docker
Full deployment with authentication, secrets management, and centralized storage. Best for teams and production environments.

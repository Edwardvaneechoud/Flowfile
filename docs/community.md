# Community

Flowfile is an MIT-licensed open source project, and it's built by the people who use it. Whether you want to ask a question, share a flow, report a bug, or propose a feature, there's a place for you here.

## Where to go

| I want to... | Go here |
|---|---|

| Discuss an upcoming release or roadmap item | [GitHub Discussions — Announcements](https://github.com/edwardvaneechoud/Flowfile/discussions/categories/announcements) |
| Share a flow, tip, or use case | [GitHub Discussions — Show and tell](https://github.com/edwardvaneechoud/Flowfile/discussions/categories/show-and-tell) |

| Contribute code or docs | [CONTRIBUTING.md](https://github.com/edwardvaneechoud/Flowfile/blob/main/CONTRIBUTING.md) |
| See what's shipped | [Releases](https://github.com/edwardvaneechoud/Flowfile/releases) |

## Discussions vs. Issues

- **Issues** are for actionable work: a bug to fix, a feature with a clear scope.
- **Discussions** are for everything else: questions, ideas in progress, release feedback, and showing off what you've built.

If you're not sure, start in Discussions — a maintainer will convert it to an issue if it fits.

## Ways to contribute

You don't have to write Python or Vue to help. Useful contributions include:

- **Answering questions** in Discussions
- **Filing clear bug reports** with a minimal reproduction
- **Improving the docs** — typos, clearer examples, missing pages
- **Sharing flows** that show off a pattern or solve a real problem
- **Testing pre-releases** and reporting back
- **Building custom nodes** — see [Create Custom Nodes](for-developers/creating-custom-nodes.md)
- **Fixing bugs or adding features** — see [CONTRIBUTING.md](https://github.com/edwardvaneechoud/Flowfile/blob/main/CONTRIBUTING.md)

## Releases

New versions land on [GitHub Releases](https://github.com/edwardvaneechoud/Flowfile/releases) and are published to [PyPI](https://pypi.org/project/flowfile/) automatically via CI. Each release has a Discussion thread for feedback — that's the right place to flag regressions, ask about migration, or request follow-ups.

## Code of conduct

Be kind and assume good intent. Flowfile follows the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). Reports of unacceptable behavior can be sent to the maintainer via GitHub.

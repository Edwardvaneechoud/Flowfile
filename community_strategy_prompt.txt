## What is Flowfile?

Flowfile is an open-source (MIT license) visual ETL (Extract, Transform, Load) platform built with a Python backend (FastAPI + Polars) and Vue.js/Electron frontend. It provides both a drag-and-drop visual flow designer AND a programmatic Python API for building data pipelines.

- GitHub: https://github.com/Edwardvaneechoud/Flowfile
- Live Demo (WASM): https://demo.flowfile.org
- Docs: https://edwardvaneechoud.github.io/Flowfile/
- Creator: Edward van Eechoud
- Current Version: 0.7.0
- Python: >=3.10, <3.14 | Node.js: 20+

## Key Differentiators

1. **Dual Interface** - Build pipelines visually OR programmatically in Python, same underlying objects
2. **Code Generation** - Export visual flows as standalone Python/Polars code (no Flowfile dependency needed)
3. **Built on Polars** - Lightning-fast, out-of-core data processing (not pandas)
4. **Visual Debugging** - Inspect data at every step
5. **Version Control Friendly** - Flows saved as human-readable YAML/JSON
6. **WASM Demo** - Try it instantly in the browser at demo.flowfile.org
7. **Embeddable** - FlowfileEditor can be embedded as a Vue component library
8. **Docker Kernel Execution** - Run custom Python code in isolated containers with any pip packages

## Target Audiences

- **Non-technical users**: Analysts, business users, Excel power users who want drag-and-drop ETL
- **Technical users**: Data engineers, data scientists, Python developers who want a Polars-based pipeline tool
- **Open-source contributors**: Developers interested in contributing to a modern data tool

## Origin Story (useful for content)

Flowfile started as a pure-Python data processing engine — the creator built a custom
FlowFile class from scratch using raw tuples, itemgetter, itertools groupby, and manual
schema inference. It handled joins, pivots, groupby, explode, and even memory optimization
all without any dataframe library. The migration to Polars was a pivotal moment that
unlocked massive performance gains while keeping the same visual-first philosophy. This
"built it from scratch, then found the right tool" journey is a compelling narrative for
blog posts and talks — it shows deep understanding of data internals and why Polars was
the right choice.
